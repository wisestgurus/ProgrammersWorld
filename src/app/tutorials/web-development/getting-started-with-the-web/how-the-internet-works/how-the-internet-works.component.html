introduction

The internet by definition is a network that connects other networks together and makes sharing of data globally possible.

The internet ensures data are delivered successfully using a set or family of protocol known as the Internet protocol suite. (Protocol means a rule or guide used in carrying out anything).

These protocols are divided into layers, but we won't go much into that. The two most important protocols we are going to focus on in this particular tutorial are; TCP/IP.

TCP stands for Transmission Control Protocol, while IP stands for Internet Protocol. They are two different protocols with each one of them having its own responsiblity or role. But they are generally classified to be one because they depend on each other alot, as you will soon find out.

Every information system shared through the internet (e.g. the web, etc) uses TCP/IP for communication and also has the ability to create its own protocol.

how TCP/IP works

  1. TCP
    • When billions of data are sent through the internet, the responsibility of TCP is to ensure that the datas are tracked and delivered successfully.
    • TCP does that by breaking down each of those data (sent by millions of people through the internet) into smaller chunks of data called packets for faster transmission (or delivery). (e.g. if a data of 1gigabyte is sent through the internet using an information system like the web, TCP breaks the data into smaller sizes (making upto 1 gigabyte) - let's say 1kb each. Hence every 1kb of data in this example is called a packet).
    • When TCP breaks the big data into packets, it assigns a TCP header to every packet. This header contains information such as how to track every packet to ensure successful delivery even when lost along the way and also how to re-arrange those packets back to their initial data, so that the receiver can now see the data exactly as it was sent.
  2. IP
    • The responsibility of IP is to ensure that those packets contain the location (or address) of both the sender and the receiver.
    • IP does that by adding its own header to each packet, known as IP header. IP header contains information about the senders (or source) address and the receivers (or destination) address.
  3. Summary
    • Tracking of data sent over the internet to ensure it is delivered successfully won't be possible without TCP.
    • Also sending the data to the right location (or address) won't be possible without IP.
    • Hence the reason why they are mostly grouped as one (TCP/IP).
TCP/IP is not the only protocol used by the internet. This is not a networking tutorial. You are learning it because, you need to have an idea of how contents on the web are shared through the internet and to be able to differentiate it with how HTTP works, which is taught in a later tutorial.
